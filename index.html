API Metadata

Metadata Element Definition
Name A human readable label for the API.
Access Point The base URI for interacting with the API.
Description A human readable description of the service functionality.
Response MIME-type A list of media types the APIs can produce. This is global to all APIs but can be overridden on specific API calls.
Documentation Documentation page URL for the API.
Terms of Service A document that describes the terms of use for the API.
Authentication Mode Lists the required security schemes to execute this operation. The object can have multiple security schemes declared in it which are all required.
Version The version of the API.
SSL Support Indication of whether SSL Support is present or absent.
Website A web page that describes the API.
Category Primary category that the API belongs to.
Publications Publication ID/URL.
API Access Restrictions Indicate whether there are restrictions to using the API
Access Point Mirrors URL of the mirror that the API is available at.
API Metadata Format Service specification format.
API Access Mode The protocols used to access the API
API Location Location, city and country of the API.
API Implementation Language Language the API was written in.
API Maturity Maturity of the API.
Social Media Links Links to social media presence such as Twitter URL, facebook, LinkedIn etc.

Service Provider Metadata

Metadata Element Definition
Responsible Organization The identifying name of the contact person/organization.
Responsible Developer Name of the developer (User ID/Name)
Contact Email An e-mail address where the provider of the service may be contacted. MUST be in the format of an email address.
Contributor Contributor person or institute.
Funding Source of funding.
Developer Forum Forum or Mailing list.

API Operation Metadata

Metadata Element Definition
Operation Title Title of the operation. A unique identifier of the operation.
Operation Description Description of the operation.
http Method The base path on which the API is served, which is relative to the host. Or a relative path to an individual endpoint. The field name MUST begin with a slash. The path is appended to the basePath in order to construct the full URL.
Authentication Mode Lists the required security schemes to execute this operation. The object can have multiple security schemes declared in it which are all required.
Operation ID An identifier/URI for the operation.
Keywords Keywords describing the operation.
Operation Documentation Documentation of the particular operation.
Path Template The path for the operation.
Access Restrictions Access restrictions to invoke the operation.
Collection Names of collections of which the resource is a part e.g. a suite, library etc.

Operation Parameter Metadata

Metadata Element Definition
Operation Parameter Name Name of the operation parameter.
Location Determines the location of the parameter.
Parameter Type Type of parameter.
Parameter Value Type Type of the value of the particular parameter.
Consumes A list of MIME types the operation can consume.
Transfer Protocol The transfer protocol of the API. Values MUST be from the list: "http", "https", "ws", "wss".
Operation Parameter Description Description of service parameter.
Min Minimum occurrence of the parameter.
Max Maximum occurrence of the parameter.
Data Format Input/Output data format.
Default Value Default value for each input/output.
Example Value Example value for each input/output.

Operation Response Metadata

Metadata Element Definition
Response Format A list of MIME types the APIs can consume.
Response Header A list of headers that are sent with the response.
Example Response Value An example of the response value.
Response Code A container for the expected responses of an operation. The container maps a HTTP response code to the expected response. The list of possible responses as they are returned from executing this operation.
Response Description A description of the response.
